TECH STACK
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- Database: MySQL with SQLAlchemy ORM

DATABASE SCHEMA (MySQL)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Table: customers
  - id           INT AUTO_INCREMENT PRIMARY KEY
  - customer_id  VARCHAR(20) UNIQUE  e.g. CUST-0001
  - first_name   VARCHAR(100)
  - last_name    VARCHAR(100)
  - email        VARCHAR(255) UNIQUE
  - phone        VARCHAR(30)
  - street       VARCHAR(255)
  - city         VARCHAR(100)
  - state        VARCHAR(100)
  - postal_code  VARCHAR(20)
  - country      VARCHAR(100)
  - created_at   DATETIME DEFAULT CURRENT_TIMESTAMP

Table: orders
  - id           INT AUTO_INCREMENT PRIMARY KEY
  - order_id     VARCHAR(20) UNIQUE  e.g. ORD-0001
  - customer_id  INT FK → customers.id
  - product      VARCHAR(255)
  - unit_price   DECIMAL(10,2)
  - quantity     INT
  - total_amount DECIMAL(10,2)  (auto = unit_price × quantity)
  - status       ENUM('Pending','In progress','Completed')
  - created_by   VARCHAR(100)
  - order_date   DATETIME DEFAULT CURRENT_TIMESTAMP

Table: dashboard_widgets
  - id             INT AUTO_INCREMENT PRIMARY KEY
  - widget_id      VARCHAR(50) UNIQUE
  - widget_title   VARCHAR(100)
  - widget_type    ENUM('KPI','Table','Bar Chart','Line Chart',
                        'Pie Chart','Area Chart','Scatter Plot')
  - description    TEXT
  - width_cols     INT DEFAULT 2
  - height_rows    INT DEFAULT 2
  - grid_x         INT DEFAULT 0
  - grid_y         INT DEFAULT 0
  - config_json    JSON   ← stores all Data Setting fields
  - created_at     DATETIME DEFAULT CURRENT_TIMESTAMP

FASTAPI BACKEND — ALL ENDPOINTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Orders CRUD:
  GET    /api/orders          → list all orders (join customer data)
  POST   /api/orders          → create order + auto-create customer record
  PUT    /api/orders/{id}     → update order
  DELETE /api/orders/{id}     → delete order

  Auto-generate IDs:
    customer_id: CUST-{4-digit sequential number}
    order_id:    ORD-{4-digit sequential number}

  Query params for GET /api/orders:
    ?date_range=today|last7|last30|last90|alltime

Dashboard Widgets CRUD:
  GET    /api/widgets         → list all saved widgets with positions
  POST   /api/widgets         → save new widget
  PUT    /api/widgets/{id}    → update widget config or position
  DELETE /api/widgets/{id}    → delete widget

Analytics endpoints (used by widgets):
  GET /api/analytics/kpi?metric=total_orders&aggregation=count
      &date_range=alltime
      → returns { value: 100 }

  Supported metrics: total_orders, total_revenue, total_customers,
                     total_sold_quantity, unit_price, quantity,
                     total_amount
  Supported aggregations: sum, average, count
  Supported date ranges: today, last7, last30, last90, alltime

  GET /api/analytics/chart?type=pie&field=status&date_range=alltime
      → returns [{ label, value, color }]

  GET /api/analytics/table?columns=order_id,product,quantity,
      total_amount&sort=asc&page=1&limit=5
      &filter_attr=status&filter_op=eq&filter_val=Pending
      → returns { rows: [...], total: N }

Widget Config Validation:
  All * fields required. Show inline errors on Save.
  Do NOT save if required fields are missing.

After Save (widget config):
  - Side panel closes
  - Widget updates to show real data from analytics API
  - Widget title updates from "Untitled" to configured name
  - Toast: "All set! Your new widget have been added successfully!"

Delete Widget (from grid):
  - Click 🗑 → confirmation modal:
    "Are you sure you want to delete the {widget title} widget?"
    [Cancel] [Delete]
  - On confirm → remove from grid + toast: "Done! Your widget has 
    been removed"

Saving the Dashboard:
  - [Save] button → persists all widget positions + configs to DB
  - Toast: "All set! Your changes have been saved successfully!"
  - Navigates back to /orders (Dashboard tab)

Unsaved Changes Guard:
  - If user clicks [Cancel] or navigates away with unsaved changes →
    show modal: "Unsaved changes — Do you want to save your changes 
    before navigating away?" [Discard] [Save]
  - Discard → discard all changes, go back
  - Save → save then go back
